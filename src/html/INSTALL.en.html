Install RubyCocoa

Install Sequence

  • Install Ruby
  • Extract RubyCocoa
  • Install LibRuby Framework
  • Build and Install RubyCocoa
  • Done
  • Check Sample

You need to use command line shell (by Terminal Application or others) for installing.

Install Ruby

If you have installed Ruby with "--enable-shared" option already, go to next section.

If you newly install Ruby on MacOS X, use 1.6.6 or later. When installing it or earlier, it is necessary to apply a patch to source.

The following is the most typical installation procedure of Ruby 1.6.6.

% cd {anywhere}
% tar zxf ruby-1.6.6.tar.gz    # extract the tarball
% cd ruby-1.6.6
% ./configure --enable-shared  # install to /usr/local
% make ; make test
% sudo make install

At the time of configure, specify the "--enable-shared" option.

Extract RubyCocoa

extract RubyCocoa sources tarball.

% cd {anywhere}
% tar zxf rubycocoa-0.2.x.tgz

Caution! Using StuffIt, building RubyCocoa will be failed because of file name length problem.

Install LibRuby Framework

LibRuby Framework is a framework for Ruby interpreter shared library and related header files. The framework is needed by RubyCocoa and RubyCocoa applications. For installing the LibRuby Framework on "/Library/Frameworks", type:

% cd {RubyCocoa srcdir}
% ruby tool/install-libruby-frmwk.rb

When you want to install this to another location:

% ruby tool/install-libruby-frmwk.rb {anywhere path}

Build and Install RubyCocoa

Install RubyCocoa that consists of libraries and RubyCocoa Framework.

Change directory to RubyCocoa sourcedir and type:

% cd {RubyCocoa srcdir}
% ruby install.rb config
% ruby install.rb setup
% sudo ruby install.rb install

After config, you will look a message "require LibRuby Framework". If you are installing by this text seqeunce, the Framework already exists.

If you want to specify the installing location of RubyCocoa framework (default is "/Library/Frameworks"), use the config option as follows:

% ruby install.rb config --frameworks=/Networks/Library/Frameworks

Setup need too long time. It's tea time...

When install, RubyCocoa libraries will be installed. And RubyCocoa Framework will be installed RubyCocoa Framework to "/Library/Frameworks".

Done

With the above installation is completion.

  • LibRuby framework - /Library/Frameworks/LibRuby.framework
  • RubyCocoa framework - /Library/Frameworks/RubyCocoa.framework
  • RubyCocoa libraries A - {site_ruby}/1.6/{archdir}/osx_objc.bundle
  • RubyCocoa libraries B - {site_ruby}/1.6/osx/*.rb
  • RubyCocoa libraries B - {site_ruby}/1.6/osx/objc/*.rb

was installed other than a Ruby itself with procedure to this place.

Check Sample

Now go to the sample dir, and try to execute some scripts.

% cd {RubyCocoa srcdir}/sample
% ruby fontnames.rb # fontname print to stdout.
% ruby sndplay.rb   # system sounds play in order.
% ruby sndplay2.rb  # system sounds play in order with short interval.

OK? Next try to execute some RubyCocoa applications. First one:

% cd {RubyCocoa srcdir}/sample
% open SimpleApp1.app

Next one is Makefile based. Type to build: 

% cd {RubyCocoa srcdir}/sample/mk_app1
% make

And launch application:

% open SimpleApp1.app

or double click 'SimpleApp1' on Finder. Next one is Project Builder based. type to build:

% cd {RubyCocoa srcdir}/sample/pb_app1
% pbxbuild

Of course you can build and run by Project Builder. Launch application:

% open build/pb_app1.app
